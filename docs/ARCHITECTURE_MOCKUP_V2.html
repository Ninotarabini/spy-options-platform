Estrategia del Proyecto

La plataforma integra capacidades de computación en el borden (Edge) para la captura de baja latencia con la escalabilidad de la nube para la distribución de señales en tiempo real.

Bloque 1: Infraestructura

Azure Control Plane (Cloud) Terraform

Gestión centralizada de recursos escalables y seguridad corporativa.

main.tf Provisionamiento de VNet, SignalR y Key Vault para seguridad de credenciales.
Static Hosting global para el Dashboard de visualización avanzada.
On-Premises Data Plane (Edge) K3s / Helm

Procesamiento local crítico para la integración directa con la API de IBKR.

K8s Arquitectura de pods distribuida para Backend, Frontend y Detector de anomalías.
Helm Versión 1.0.0 desplegada. Gestión de configuraciones mediante valores dinámicos.
Sistema de Ejecución (Trading Bot) Experimental

Módulo en modo observación. Pendiente de verificación de fiabilidad para operaciones automáticas.

Bloque 2: Flujo de Datos

Ruta de Baja Latencia: IBKR -> IB Gateway -> Detector (AsyncIO) -> SignalR -> UI.

Principio Broadcast-First: Se prioriza la emisión del evento sobre la persistencia en base de datos para ganar ~200ms de tiempo de respuesta visual.

Operaciones y Seguridad

Pipeline CI/CD

Build & Scan automático en cada push. Seguridad gestionada por Trivy.

Monitoreo Híbrido

Prometheus (Local) + Application Insights (Nube).
