01 Estrategia Híbrida

La arquitectura del proyecto SPY Options rompe el paradigma tradicional de "todo en la nube" para priorizar la **baja latencia**. El procesamiento pesado de opciones ocurre en servidores locales (Edge) directamente conectados al gateway de IBKR, liberando a la infraestructura de Azure para tareas de gestión, seguridad y distribución masiva de señales.

04 Flujo de Datos Actual

Lógica de "Broadcast-First"

A diferencia de sistemas tradicionales que primero guardan en DB y luego notifican, el Backend de SPY Options ejecuta un **Broadcast Asíncrono** inmediato a SignalR.

Secuencia de Ejecución (Auditada):

  1. Recibe POST de anomalía/flow del Detector.
  2. Dispara Broadcast a SignalR (Websocket) -> Dashboard Actualizado.
  3. Inicia BackgroundTask para persistencia en Azure Table Storage.
  4. Responde 201 Accepted al emisor.
Hybrid Link Conectividad IPsec VPN Site-to-Site (100Mbps). Tráfico cifrado entre el cluster local y el VNet de Azure.

05 DevOps & Workflow CI/CD

Todo el ciclo de vida se gestiona mediante GitHub Actions, garantizando que el diseño del repositorio sea la única fuente de verdad.

Seguridad: Trivy

Escaneo de vulnerabilidades en cada build de imagen. Bloqueo automático de CVEs críticos antes del push a ACR.

Frontend a SWA, Infra a Azure via Terraform, y Apps a K8s via Helm (Manual/Trigger).

06 Observabilidad Híbrida

Local Stack Prometheus + Grafana detallado. Captura métricas de hardware, pods y latencia de red interna.
Azure Application Insights para monitoreo de nivel de aplicación, telemetría de fallos HTTP y logs agregados.
Logs Fluentd DaemonSet tailing logs de pods y enviándolos a Azure Log Analytics.
